# i18n Multilingual Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add full multilingual support (Simplified Chinese + Japanese) to MapQuiz.pro using next-intl, covering UI, quiz metadata, SEO, and place names.

**Architecture:** App Router routes move into a `[locale]` dynamic segment; `next-intl` middleware handles locale detection and redirects; translations live in `messages/{en,zh,ja}.json`; place name localizations are static TypeScript maps loaded at render time.

**Tech Stack:** next-intl ^3.x, Next.js 16 App Router, Noto Sans SC / Noto Sans JP (Google Fonts for CJK)

## Task 1: Install next-intl and configure routing

**Files:**
- Create: `src/i18n/routing.ts`
- Create: `src/i18n/request.ts`
- Create: `src/middleware.ts`
- Modify: `next.config.ts`

- [ ] **Step 1: Install next-intl**

```bash
pnpm add next-intl
```

Expected: `next-intl` appears in `package.json` dependencies.

- [ ] **Step 2: Create i18n routing config**

Create `src/i18n/routing.ts`:

```ts
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['en', 'zh', 'ja'],
  defaultLocale: 'en',
  localePrefix: 'as-needed'
});
```

- [ ] **Step 3: Create server request config**

Create `src/i18n/request.ts`:

```ts
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  const locale = (await requestLocale) ?? routing.defaultLocale;
  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default
  };
});
```

- [ ] **Step 4: Create middleware**

Create `src/middleware.ts`:

```ts
import createMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';

export default createMiddleware(routing);

export const config = {
  matcher: ['/((?!api|_next|.*\\..*).*)']
};
```

- [ ] **Step 5: Add next-intl plugin to next.config.ts**

Replace current `next.config.ts` content:

```ts
import createNextIntlPlugin from 'next-intl/plugin';
import type { NextConfig } from 'next';

const withNextIntl = createNextIntlPlugin('./src/i18n/request.ts');

const nextConfig: NextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'api.mapbox.com',
        pathname: '/styles/v1/mapbox/**'
      }
    ]
  }
};

export default withNextIntl(nextConfig);
```

- [ ] **Step 6: Commit**

```bash
git add src/i18n/ src/middleware.ts next.config.ts pnpm-lock.yaml package.json
git commit -m "feat(i18n): install next-intl and configure routing"
```

  if (!props.initialConfig?.countryCode) {

      ? require('@/lib/data/regions/locales/zh/jp').zhJpNames
      : require('@/lib/data/regions/locales/ja/jp').jaJpNames;
  }
  return null;
}, [locale, props.initialConfig?.countryCode]);

In `src/app/sitemap.ts`, wrap each URL to also emit `/zh/` and `/ja/` variants:

```ts
import { routing } from '@/i18n/routing';
import { quizTopics } from '@/lib/data/quizTopics';
import type { MetadataRoute } from 'next';

const SITE_URL = 'https://mapquiz.pro';

export default function sitemap(): MetadataRoute.Sitemap {
  const staticRoutes = ['', '/quizzes'];
  const slugRoutes = quizTopics.map((t) => `/quiz/${t.slug}`);
  const allRoutes = [...staticRoutes, ...slugRoutes];

  return allRoutes.flatMap((route) =>
    routing.locales.map((locale) => ({
      url: locale === routing.defaultLocale
        ? `${SITE_URL}${route}`
        : `${SITE_URL}/${locale}${route}`,
      lastModified: new Date(),
      changeFrequency: 'weekly' as const,
      priority: route === '' ? 1 : 0.8
    }))
  );
}
```
