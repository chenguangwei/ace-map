# i18n 多语言支持设计文档

**日期**：2026-04-20  
**范围**：全量翻译（UI + Quiz 题目 + SEO 元数据 + 地名）  
**目标语言**：简体中文（zh）、日文（ja），英文（en）为默认语言，架构可扩展  
**方案**：next-intl + App Router 路径前缀  

---

## 1. 路由结构

### 目录重构

所有页面路由移入 `src/app/[locale]/` 动态段：

```
src/app/
├── [locale]/
│   ├── layout.tsx          ← locale-aware 根布局
│   ├── page.tsx            ← 首页（从 src/app/page.tsx 移入）
│   ├── quizzes/page.tsx    ← 题库页（移入）
│   ├── quiz/[slug]/page.tsx← 题目详情页（移入）
│   └── game/page.tsx       ← 游戏页（移入）
├── globals.css             ← 保留在外
└── favicon.ico 等静态资源  ← 保留在外
```

### URL 规则

| Locale | 路径示例 |
|--------|---------|
| en（默认） | `/quizzes`、`/quiz/world-countries` |
| zh | `/zh/quizzes`、`/zh/quiz/world-countries` |
| ja | `/ja/quizzes`、`/ja/quiz/world-countries` |

使用 `localePrefix: 'as-needed'`，英文不加前缀。

### Middleware

新增 `src/middleware.ts`：

```ts
import createMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';

export default createMiddleware(routing);

export const config = {
  matcher: ['/((?!api|_next|.*\\..*).*)']
};
```

### i18n 路由配置

新增 `src/i18n/routing.ts`：

```ts
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['en', 'zh', 'ja'],
  defaultLocale: 'en',
  localePrefix: 'as-needed'
});
```

新增 `src/i18n/request.ts`（Server Component 侧 locale 注入）：

```ts
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  const locale = (await requestLocale) ?? routing.defaultLocale;
  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default
  };
});
```

## 3. 地名翻译

### 数据结构

`PlaceItems` 的 `name` 字段保持英文不变（作为唯一 key）。新增本地化映射层：

```
src/lib/data/regions/locales/
├── zh/
│   ├── world.ts   ← Record<string, string>，英文名 → 中文名
│   ├── jp.ts
│   ├── de.ts
│   └── ...（按需覆盖，未覆盖的回退到英文）
└── ja/
    ├── world.ts
    ├── jp.ts
    └── ...
```

示例（`zh/world.ts`）：

```ts
export const zhWorldNames: Record<string, string> = {
  'Japan': '日本',
  'China': '中国',
  'United States': '美国',
  'Germany': '德国',
  // ...
};
```

### 运行时替换

新增 `src/lib/utils/localizePlace.ts`：

```ts
export function localizePlace(name: string, locale: string, map: Record<string, string>): string {
  if (locale === 'en') return name;
  return map[name] ?? name; // 未找到时回退英文
}
```

地名映射在游戏渲染层（`GameBar`、`Main` 等组件）按 locale 懒加载并注入，不改动底层 `PlaceItems` 数据。

## 4. Server Components 集成

```tsx
// src/app/[locale]/page.tsx
import { setRequestLocale } from 'next-intl/server';
import { getTranslations } from 'next-intl/server';

export default async function Page({ params }) {
  const { locale } = await params;
  setRequestLocale(locale);
  const t = await getTranslations('HomePage');

  return <h1>{t('headline')}</h1>;
}
```

`[locale]/layout.tsx` 中调用 `setRequestLocale(locale)` 一次即可，子页面继承。

### SEO 元数据

```tsx
export async function generateMetadata({ params }) {
  const { locale, slug } = await params;
  const t = await getTranslations({ locale, namespace: 'QuizTopics' });
  return {
    title: t(`${slug}.seoTitle`),
    description: t(`${slug}.seoDescription`),
    alternates: {
      canonical: `https://mapquiz.pro/${slug}`,
      languages: {
        'zh': `https://mapquiz.pro/zh/quiz/${slug}`,
        'ja': `https://mapquiz.pro/ja/quiz/${slug}`,
      }
    }
  };
}
```

`next-intl` 自动在 `<html>` 标签注入正确的 `lang` 属性。

## 5. Client Components 集成

```tsx
// src/lib/components/game/GameBar.tsx
'use client';
import { useTranslations } from 'next-intl';

const t = useTranslations('GameBar');
<span>{t('time')}</span>
```

`[locale]/layout.tsx` 通过 `NextIntlClientProvider` 将翻译 messages 传递给客户端树。

## 6. 语言切换器

在 Navbar 中新增 `LocaleSwitcher` 组件：

```tsx
'use client';
import { useRouter, usePathname } from 'next-intl/navigation';

export function LocaleSwitcher() {
  const router = useRouter();
  const pathname = usePathname();
  
  return (
    <select onChange={(e) => router.replace(pathname, { locale: e.target.value })}>
      <option value="en">EN</option>
      <option value="zh">中文</option>
      <option value="ja">日本語</option>
    </select>
  );
}
```

切换时保留当前路径，无需整页刷新。

## 7. 字体支持

当前使用 `Geist` 字体，对中文/日文显示效果有限。在 `[locale]/layout.tsx` 中根据 locale 条件加载：

- `zh`：加载 `Noto Sans SC`（Google Fonts）
- `ja`：加载 `Noto Sans JP`（Google Fonts）
- `en`：继续使用 `Geist`

---

## 8. 不在范围内

- 繁体中文（可后续通过加 `zh-TW` locale 扩展）
- 韩文、法文等其他语言
- 用户自定义语言偏好持久化（依赖浏览器 `Accept-Language` 或用户选择即可）
- 机器翻译自动化流水线（翻译内容手动维护）
